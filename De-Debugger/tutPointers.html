References

A reference variable is an alias for another object. It is created using the & sign.

Two things to note:

  • Anything done to the reference also happens to the original.
  • Aliases cannot be changed to alias something else.

Pass-By-Reference

In C++, pass-by-reference refers to passing parameters to a function by using references.

It allows the ability to:

  • Modify the value of the function arguments.
  • Avoid making copies of a variable/object for performance reasons.

Common Errors

Avoid common pitfalls when using C++ pointers and references

Null Pointers

A null pointer is a pointer that does not point to a valid memory location. Dereferencing a null pointer can lead to program crashes or undefined behavior. To avoid null pointer errors, make sure your pointer is initialized to a valid memory location before dereferencing it.


Dangling Pointers

A dangling pointer is a pointer that points to a memory location that has been deallocated or freed. Dereferencing a dangling pointer can lead to program crashes or undefined behavior. To avoid dangling pointer errors, make sure your pointer is updated or set to null when the memory it points to is deallocated or freed.


Uninitialized Pointers

An uninitialized pointer is a pointer that has not been assigned a value. Dereferencing an uninitialized pointer can lead to program crashes or undefined behavior. To avoid uninitialized pointer errors, make sure your pointer is assigned a valid memory location before dereferencing it.


Invalid References

An invalid reference is a reference that does not refer to a valid object or variable. Using an invalid reference can lead to program crashes or undefined behavior. To avoid invalid reference errors, make sure your reference is initialized to a valid object or variable before using it.


References to Temporary Objects

A reference to a temporary object is a reference that refers to an object that is destroyed at the end of the statement or expression it is used in. Using a reference to a temporary object can lead to program crashes or undefined behavior. To avoid references to temporary object errors, make sure your reference refers to an object with a longer lifetime.
